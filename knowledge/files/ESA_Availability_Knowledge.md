# ESA Capability Availability Knowledge

Use this document as the authoritative knowledge source for capability availability questions.
Availability means capability status, sub-capability status, and regional availability.
It does not mean uptime, resilience, hosting, or infrastructure unless explicitly requested.

## Broad Overview

- Query anchors: tell me about capability availability; provide me with a capability availability summary; give me a capability availability summary; capability availability summary; live; externally marketable; internal; capabilities; capability families; regions
- Summary: Use the status_overview, family_overview, and region_overview rows to answer broad capability availability questions. Prioritize explicit spreadsheet status and region evidence, then drill into capability families, capabilities, sub-capabilities, and regions.

## Capability To Region Query Anchors

- Query anchors: where is invoicing live; in which regions is invoicing live; where is accounting live; where is tax filing externally marketable; where is transaction emissions internal; where is <capability> live; in which regions is <capability> externally marketable; in which regions is <capability> internal
